Question 1: Can you describe how the widget looks, such as its colors, size of elements, and layout? Why do you make these design choices?

    The component is a rectangular banner with rounded corners, positioned horizontally near the top of the screen. It features a deep navy blue background (#002347) with crisp white text for high readability. The layout, managed by flexbox, places the descriptive text on the left, a call-to-action button on the right, and a close icon at the far right. The call-to-action button uses a warm, muted gold color (#D4A017) to attract attention without creating alarm. Text size is set to a readable 16px. These choices create a professional, calm, and trustworthy appearance. The sophisticated color palette avoids the aggressive red/yellow often used in high-pressure sales, helping users feel in control and not manipulated, which aligns with design rules for fostering trust and ensuring user comfort. A subtle box-shadow lifts the banner slightly, integrating it smoothly with the page. On mobile, the layout stacks vertically for better readability on narrow screens.

    Question 2: What triggers the appearance of this widget?

    The banner is designed to appear automatically on page load. It is not triggered by a specific user action like scrolling or hovering. This is a common and predictable behavior for site-wide announcements, ensuring that the information is presented upfront without surprising the user. The banner would typically be configured to show on key entry pages like the homepage, category pages, or all pages, depending on the scope of the sale. To respect the user's space, it would also be configured to be dismissible and not reappear during the same browsing session after being closed, using a cookie or session storage. This predictable pattern fosters trust by ensuring the system behaves as expected and is not intrusive.
